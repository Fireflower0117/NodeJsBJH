REST와 라우팅 사용

서버에 요청을 보낼떈 주소를 통해 요청의 내용을 표현한다. 예를들어 /index.html을 요청하면
index.html파일을 보내달라는 뜻이다.

항상 html만 요청할 필요는 없다.  css, js, png등 여러가지 파일을 요청할수도있고
특정 동작(행위)를 주소정보를 통해 요청할수도있다. 여기서 REST가 등장한다.

REST는 REpresentational State Transfer의 줄임말로 서버에 자원을 정의/요청하는 것이다 .
서버와 일종의 약속이라고 봐도 된다. REST 약속에 대한 정보가 REST API이다.

REST API에는 많은 규칙이 있다. 여기서 전부 나열하지는 않고 몇가지 규칙만 알아본다.

규칙중 한가지는 주소는 의미를 명확히 전달하기위해 명사로 구성한다.
이를테면 /user라면 사용자에대한 요청이고, /post라면 게시글에 대한 요청이다.
단순히 명사만 있으면 무슨동작을 하라는것인지 알기 어려우니, 주소정보되 HTTP요청 메소드라는것을 사용한다.

HTTP요청 메소드 종류
GET     : 서버자원을 가져오고자 할떄 사용한다.
POST    : 서버에 자원을 새로 등록할떄 사용한다
PUT     : 서버의 등록된 자원을 요청한 자원으로 치환하고자 할떄 사용한다.
PATCH   : 서버 자원의 일부만 수정하고자 할떄 사용한다. 요청본문에 수정할 데이터를 넣어 보낸다.
DELETE  : 서버의 자원을 삭제할떄 사용한다.
OPTIONS : 요청을 하기전에 통신옵션을 설정할떄 사용한다.

주소하나가 여러가지 요청 메소드를 가질수있다.
예를들어 GET으로 /user를 요청하면 사용자 정보를 조회하겠다는것이고
POST로 /user를 요청하면 사용자를 등록하겠다는 의미로 해석할수있다.

이렇듯 REST는 주소정보와, 요청메소드만으로도 어떤 요청인지를 알수있는것이 장점이다.
또한 HTTP통신을 사용하면 클라이언트가 누구든 상관없이 같은 방식으로 서버와 소통할수있다.
즉 서버와 클라이언트는 분리되어있다는 뜻이다.  서버와 클라이언트를 분리하면 서버의 기능을 확장할떄
클라이언트에 구애되지 않아 좋다.
